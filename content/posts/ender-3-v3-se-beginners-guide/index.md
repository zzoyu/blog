# 용어 사전

기기 자체에서 쓰이는 용어들도 있고, 제조사 특징도 있는 등 이 분야는 일단 진입장벽이 꽤 높다고 느꼈습니다. 국내외 3D 프린터 커뮤니티에서 자주 언급되지만 생소할만한 용어들을 정리해보겠습니다.

당장 이것들을 다 외워야 하는 것은 아니지만, 분명 얼마 지나지 않아 마주치게 될 것입니다.

그냥 이런게 있구나 정도로 슥슥 넘어가면 됩니다.

## 1. 소프트웨어 관련 용어

### STL, STEP, GCode, 3MF

처음으로 관련 커뮤니티나 디자인 공유 사이트에 들어가면 저런 파일로 한가득이라 각각이 무엇인지 당췌 알기가 어렵습니다.

- STL : 웬만한 3D 설계, 모델링 프로그램이 지원하는 표준 3D 모델링 포맷입니다. 기본적인 설계도만 있는 상태라고 생각하면 되겠습니다. 모델링만 공유하는 경우에는 가장 많이 사용되는 원시적인 포맷입니다.
- STEP : STL처럼 표준 3D 모델링 포맷입니다. CAD 프로그램간 호환을 위해 만들어진 규격이라 STL에 비해 수정 자유도가 더 높지만, 이 때문에 공개하지 않는 업로더가 많습니다.
- GCode : 3D 프린터에서 사용하는 명령어의 집합입니다. 프린터 버전 응용프로그램 정도로 이해하면 되겠습니다. 여러 명령을 순차적으로 실행하는데, 가열, 이동, 압출 등의 동작을 포함합니다. 즉, 실제로 기계가 어떻게 움직여야 하는지를 기술하는 파일입니다. 이 파일로 실제 프린팅을 시작하게 됩니다.
- 3MF : 슬라이서 프로그램들이 지원하는 포맷입니다. 3D 프린터의 설정이나 3D 모델, 데이터의 배치 정보를 포함하고 있습니다. 3MF 파일의 업로더와 같은 기종의 프린터를 사용하는 경우 별도 설정 없이 바로 출력이 가능합니다.

즉, 데이터의 흐름은 모델링 프로그램 -> STL/STEP -> 슬라이서(3MF) -> GCode -> 프린터 순으로 이어지게 됩니다.

### 슬라이서 / Slicer

지금 당신이 설치해야 할 것.

3D 프린터는 기본적으로 적층형입니다. 얇은 층을 쌓아올려 모델을 완성합니다. 따라서 "모델"을 "얇은 층"의 집합으로 변환해야 합니다. 이 역할을 해주는 것이 슬라이서입니다. 이렇게 보면 별 것 아닌 것 같이 느껴지지만 GCode를 만드는 주체이기 때문에 사실상 기기의 움직임을 구체적으로 지휘하는 핵심 부분입니다. 그러므로 기기에 대한 설정이나 필라멘트에 대한 설정도 이 슬라이서 프로그램에 갖춰져 있어야 합니다. 얼마나 빠르게 움직일지, 얼마나 뜨겁게 가열할지 같은 부분도 GCode에 포함되기 때문입니다.
`Creality Print`, `Ultimaker Cura`, `OrcaSlicer` 등등 다양한 프로그램이 존재하지만 보통은 저 셋 중 하나를 골라잡게 됩니다.

reddit 내의 이 프린터 커뮤니티([r/Ender3V3SE](https://www.reddit.com/r/Ender3V3SE/))에서는 `OrcaSlicer`를 추천하는 경우가 많습니다. 저는 `OrcaSlicer`의 단점을 훨씬 많이 느끼고 있어서 (메쉬 레벨링 활성화 코드 누락이라든지, UI 잔버그라든지) `Cura`를 써보려 합니다. 그냥 본인 마음에 드는걸 쓰시기 바랍니다... 뭘 쓰든 설정하게 되는 값들은 큰 차이가 없고, 진행 절차도 동일하기 때문에 마음에 들지 않으면 바꾸면 됩니다.

### Marlin? Klipper?

3D 프린터 펌웨어의 기반이 되는 프로그램입니다. 기본적으로 대부분 시중에 판매되는 프린터의 스톡 펌웨어는 `Marlin` 기반인 것 같습니다. 어찌되었든 둘 다 GCode를 해석해 기계를 동작시키는 역할을 합니다. 마이크로프로세서 특성상, `Marlin`은 한계가 어느정도 있기 때문에 `Klipper`로 넘어가는 경우가 종종 있습니다. 이에 대해 조금 더 알고 싶다면 이 [아티클](https://all3dp.com/2/klipper-vs-marlin-difference/)을 참고하세요. 솔직히 지금 단계에서 신경쓸 필요는 없고, 우리의 프린터는 일단은 `Marlin` 기반입니다. GCode를 직접 수정하거나 입력해야 하는 경우 `Marlin`의 레퍼런스를 찾으면 됩니다.

## 2. 하드웨어 관련 용어

### C13? C14?

Ender 3 V3 SE 모델에만 존재하는 개념이기 때문에 추가했습니다.
이것은 프린터 메인보드 모델 번호를 의미합니다. 시리얼넘버를 확인하면 알 수 있어요. 프린터를 까뒤집기 힘들다면 버전 정보 화면에서 확인할 수 있습니다. C13/C14를 굳이 구분하는 이유는 둘의 칩셋도 다르고(`GD303RET6`/`STM32F401`) 펌웨어 업데이트시 절차와 파일이 다릅니다. 솔직히 뭐가 더 낫다는 모르겠어요. 굳이 따지자면 C13의 칩셋이 더 좋은 것 같긴 합니다. C14는 조금 더 비싼 칩셋이긴 하지만... 이 부분은 악세사리(네뷸라 패드)나 펌웨어 업데이트 시 약간의 제약사항이 있다 정도만 확인이 되네요. 솔직히 C14 사용하는 입장에서 불편함을 느끼지는 못하고 있지만 C13의 자료가 조금 더 많고 안정적인 것 같습니다. 여하튼, C14인데 C13용 수정을 시도하거나, 그 반대의 경우 벽돌이 될 수 있으므로 본인 기기가 어떤 버전인지는 알아두는 편이 좋습니다.

### 베드 / Bed

출력물이 놓이는 평평한 판입니다. 균형이 맞는지 확인해야 합니다. 본 모델의 경우 기본적으로 균형이 틀어져있는 사례가 종종 나옵니다.

[r/ender3 ENDER 3 V3 SE factory uneven/warped bed - SOLUTION!](https://www.reddit.com/r/ender3/comments/181mriz/ender_3_v3_se_factory_unevenwarped_bed_solution/)

### 겐트리 / Gantry

프린터에 달려있는 큰 기둥 그거. 겐트리 정렬이 잘못되는 경우가 실제로 종종 있어 앞뒤로 기울어 있는지 확인하는 것이 필요합니다.

### 핫엔드 / Hotend

필라멘트가 녹을 수 있도록 가열되는 부분입니다. 당연히 매우 뜨거우므로 핫엔드를 항상 조심해야 합니다. 실리콘 양말이 씌워져 있는데, 온도를 최대한 유지시켜주는 부분입니다. 재수가 없으면 핫엔드가 필라멘트 덩어리로 뒤덮이는 대참사가 발생하기도 합니다.

### 노즐 / Nozzle

가열된 필라멘트가 나오는 부분입니다. 펜 심 같은 역할입니다. 노즐에 이물질이 들어가거나 막히지 않도록 주의해야 합니다.
보통 0.4mm 노즐이 장착되어 있고, 이걸로 0.2레이어 출력을 하는 것이 일반적입니다.

### CFS, AMS

다양한 모델 공유 사이트를 돌아다니다보면 `CFS`나 `AMS`라는 단어를 자주 접하게 됩니다. 대체로 모델을 다운로드 받을 때 `AMS Ver`와 `No AMS Ver`로 나뉘어 있는 경우가 있습니다. 필라멘트 공급 장치의 명칭으로 여러가지 색을 표현할 때 사용합니다. 그냥 알아만 두고 신경쓰지 않도록 합니다. 다양한 색을 출력하고 싶다면 "일시 정지" 후 "필라멘트 교체"를 통해 진행합니다.

## 3. 캘리브레이션이나 그 외 설정 관련 용어

### "안착" / Bed Adhesion

출력물이 베드에 잘 붙어있는 상태를 "안착"이라고 표현합니다. 안착이 잘 되지 않으면 인쇄 중에 출력물이 베드에서 떨어지게 되는데, 이렇게 되면 사실상 되살리기 어렵습니다. 따라서 장시간 인쇄가 이어져도 베드에 잘 붙어있도록 하는 것이 중요합니다.

### 레벨링 / Leveling

## 펌웨어 업그레이드

일단 본인 기기의 펌웨어가 최신인지 확인해야 합니다.

펌웨어 업그레이드는 간단하게는 SD카드를 이용하는 방법과 USB 데이터 케이블로 연결된 시스템을 이용하는 방법이 있습니다. 그리고 프린터 자체 펌웨어와 LCD 펌웨어가 따로 존재합니다. LCD 펌웨어는 업그레이드 하지 않아도 사용에 지장은 없습니다. 파일은 각각 `.bin` 파일과 `.tft` 파일입니다. Creality 홈페이지에서 다운로드 가능합니다.

## 필요한 모델을 직접 만드는 방법?

모델링에는 여러가지 방법이 있지만, 쉽고 간편한 방법을 소개하겠습니다.
PC에서 작업하는 방법과 iPad에서 작업하는 방법입니다.
그리고 개발자에게 조금 더 친숙한 방법도 소개합니다.

### PC

사용 용도에 따라 크게 갈리기 때문에 대충 프로그램들 리스트만 적겠습니다.

- Fusion 360 (무료/유료) : 많이들 쓰는 프로그램.
- FreeCAD (무료) : 오픈소스 CAD 프로그램. 설계에 특화되어 있음
- Blender (무료) : 설계보단 모델링에 조금 더 특화
- OpenSCAD (무료) : 스크립트 기반 모델링. 간단하고 단순하여 개발자에게 친숙함.
- Tinkercad (무료) : 직관적이고 쉬움.

### iPad

- Shapr3D (유료) : 정말 잘 만들었다고 생각하지만 유료이며, 전문가용 도구에 걸맞는 가격이기 때문에 당장 추천하지는 않습니다. 무료 버전이 있으나 저장 시 기능 제한이 있으므로 시도 전에 확인하시기 바랍니다.
- Onshape : 공개 프로젝트에 한해서 무료 이용이 가능하나, 인터넷 연결이 필요하여 오프라인에서 작업이 불가능합니다.
- Tinkercad (추천) : 직관적이라 처음 사용하는 사람도 금방 익숙해질 수 있습니다. 우선은 Tinkercad를 추천합니다.

### Coding, Scripting

- OpenSCAD : 스크립트 기반 모델링. 간단하고 단순하여 개발자에게 친숙함.
- CadQuery : Python 기반 모델링. OpenSCAD보다 조금 더 복잡하지만, Python을 다룰 줄 안다면 훨씬 더 편리하게 모델링이 가능합니다.
- Build123d : CadQuery를 기반으로 만들어진 Python 라이브러리. CadQuery보다 더 직관적이고 간단하게 모델링이 가능합니다. 3D 프린터를 다루는 개발자에게는 가장 추천하는 방법입니다.

# 메인터넌스

## HW

### 윤활

실리콘 오일이나 윤활 그리스, 방청윤활제 등으로 기계가 움직이는 부분들을 주기적으로 관리합니다. 그냥 자전거 체인에 기름칠하는 정도의 느낌으로,, Super Lube 제품을 사용하는 사람도 많지만 솔직히 WD-40도 문제는 없습니다. WD-40은 녹을 방지할 겸사겸사 윤활도 하는 제품입니다. 다만 저는 스프레이형 제품보다는 실리콘오일이 다루기가 쉬운 것 같습니다.

### 빌드 플레이트

중성 세제(주방세제)로 세척. 부드러운 수세미 사용. 건조.

또는 플레이트에 지장이 없다면 IPA(이소프로필 알코올)로 닦기. 쿠팡 등지에서 이소프로필 알콜스왑을 구매하면 굉장히 편리합니다.

### 필라멘트

필라멘트는 건조한 환경에서 보관해야 합니다. 출력물 문제가 생기면 가장 큰 원인 중 하나가 습기먹은 필라멘트입니다.
단, PLA는 그렇게 습기에 민감하지 않습니다. 어쨌든 필라멘트는 그냥 3.9L~4L 시리얼통에 필라멘트를 넣고 실리카겔 좀 넣어두면 됩니다.

## FAQ
